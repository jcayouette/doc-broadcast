1. Overview

Entering the {susemgr} URL in a browser takes you to the Sign In  ] screen. If you click the menu:About[ link before logging in, you will find documentation links, including a search function, and the option to request your login credentials if you forgot either password or login. Click Lookup Login/Password for retrieving or resetting your login information. For more information, see User Account.

After logging into the Web interface of {susemgr} , the first page to appear is the Overview  ] page of the menu:Home[ menu. The Overview page contains important information about your systems, including summaries of system status, actions, and patch alerts.

If you are new to the {susemgr} Web interface, read [_ref.webui.intro] to familiarize yourself with the layout and symbols used throughout the interface.
Figure 1. Overview

Overview  ] is split into functional sections  with the most critical sections displayed first. Users can control which of the following sections are displayed by making selections on the menu:Home[Your Preferences page. Refer to Your Preferences for more information.

  • The Tasks  ] section lists the most common tasks an administrator performs via the Web interface. Click any link to reach the page within {susemgr} that allows you to accomplish that task. +image::home_overview_tasks_pane.png[

  • If any systems have not been checking in to {susemgr} , they are listed under Inactive Systems  ] to the right. Highlighting them in this way allows an administrator to quickly select those systems for troubleshooting. +image::home_overview_inactive_systems_pane.png[

  • The Most Critical Systems  ] section lists the most critical systems within your organization. It provides a link to quickly view those systems and displays a summary of the patch updates that have yet to be applied to those systems. Click the name of the system to see its menu:System Details[ page and apply the patch updates. Below the list is a link to View All Critical Systems  ] on one page. +image::home_overview_critical_systems_pane.png[

  • The Recently Scheduled Actions  ] section lists all actions less than thirty days old and their status: failed  completed  or pending. Click the label of any given action to view its details page. Below the list is a link to menu:View All Scheduled Actions[ on one page, which lists all actions that have not yet been carried out on your client systems. +image::home_overview_pending_actions_pane.png[]

  • The Relevant Security Patches  ] section lists all available security patches that have yet to be applied to some or all of your client systems. It is critical that you apply these security patches to keep your systems secure. Below this list find links to all patches (menu:View All Patches[ ) and to the patches that apply to your systems (View All Relevant Patches  ] ). +image::home_overview_latest_errata_pane.png[

  • The System Group Name  ] section lists groups you may have created and indicates whether the systems in those groups are fully updated. Click the link below this section to get to the menu:System Groups[ page, where you can choose System Groups  ] to use with the System Set Manager. +image::home_overview_system_groups_pane.png[

  • The Recently Registered Systems  ] section lists the systems added to the {susemgr} in the past 30 days. Click a system’s name to see its menu:System Details[ page. Click the link below this section to View All Recently Registered Systems  ] on one page. +image::home_overview_recently_registered_pane.png[

To return to this page, select Home  Overview on the left bar that is also known as “the menu” .

2. Notification Messages

The Home  Notification Messages page allows you to manage your notification messages of the {susemgr} server. image::home_notification_messages.png[]

The Notification Messages  ] page displays two tabs (menu:Unread Messages[ and All Messages ) with a list of all your collected messages. Several columns provide information for each message:

  • Severity : The severity levels “info” , “warning” , and “error” are available. For each failure a customized button (in line with the message) is available to react to that failure.

  • Type : Available types are:

    • Onboarding failed(“error” )

    • Channel sync finished (“info” )

    • Channel sync failed (“error” )

  • Description : The text of the message with a link to the channel.

  • Created : The date when the message was created.

  • Action Read|Delete :

    • By clicking the letter icon a message can be flagged as “read” or “unread” .

    • By clicking the waste bin icon the message will be deleted immediately.

You can sort the messages by clicking a column label of the list header line.

3. User Account

3.1. Your Account

Modify your personal information, such as name, password, and title from the Home  User Account  Your Account page. To modify this information, make the changes in the appropriate text fields and click the Update  ] button at the bottom. image::home_user_account_user_details.png[

If you change your {susemgr} password, for security reasons you will not see the new password while you enter it. Replace the asterisks in the Password  ] and menu:Confirm Password[ text fields with the new password.

If you forget your password or username, go to the sign in screen and click the About  ] link  then select the menu:Lookup Login/Password[ page. Here you can either specify your login and email address or only your email address if you are not sure about the username. Then click Send Password  ] or menu:Send Login[ respectively.

3.2. Addresses

On the Addresses  ] page you can manage your mailing  billing and shipping addresses  and associated phone numbers. image::home_user_account_addresses.png[

Click Fill in this address  ] or menu:Edit this address[ below the address to be modified or added, make your changes, and click Update  ] . image::home_user_account_edit_address.png[

3.3. Change Email

The email address listed in the Your Account  ] page is the address to which {susemgr} sends email notifications if you select to receive patch alerts or daily summaries for your systems on the menu:Your Preferences[ page. image::home_user_account_change_email.png[]

To change your preferred email address, click Change Email  ] in the left navigation bar. Enter your new email address and click the menu:Update[ button. Invalid email addresses like those ending in @localhost are filtered and rejected.

3.4. Credentials

View or enter external system or API credentials associated with your {susemgr} account. image::home_user_account_credentials.png[]

3.5. Account Deactivation

The Account Deactivation  ] page provides a means to cancel your {susemgr} service. To do so  click the menu:Deactivate Account[ button. The Web interface returns you to the login screen. If you attempt to log back in, an error message advises you to contact the {susemgr} administrator for your organization. image::home_user_account_account_deactivation.png[]

{susemgr}Administrator Account

If you are the only {susemgr} Administrator for your organization, you cannot deactivate your account.

4. Your Preferences

The Home  Your Preferences page allows you to configure {susemgr} options. image::home_your_preferences.png[]

  • Email Notifications — Determine whether you want to receive email every time a patch alert is applicable to one or more systems in your account.

    IMPORTANT: This setting also enables Management and Provisioning customers to receive a daily summary of system events. These include actions affecting packages, such as scheduled patches, system reboots, or failures to check in. In addition to selecting this check box, you must identify each system to be included in this summary email. By default, all Management and Provisioning systems are included in the summary. Add more systems either individually on the System Details  ] page or for multiple systems at once in the menu:System Set Manager[ interface. Note that {susemgr} sends these summaries only to verified email addresses. To disable all messages, simply deselect this check box.

  • SUSE Manager List Page Size  ] — Maximum number of items that appear in a list on a single page. If more items are in the list  clicking the menu:Next[ button displays the next group of items. This preference applies to system lists, patch lists, package lists, and so on.

  • menu:"Overview" Start Page[] — Select the information sections that are displayed on the Overview  ] page of the menu:Home[ menu. Check the box to the left of the information section you would like to include.

  • Locale Preferences  ] - Select and set your {susemgr} interface to your local time by selecting the appropriate menu:Time Zone[ from the drop-down box, then click the Save Preferences  ] button to apply the selection. +image::home_locale_preferences.png[

  • CSV Files  ] — Select the separator character to be used in downloadable CSV files. menu:Comma[ is the default; as an alternative use Semicolon , which is more compatible with Microsoft Excel.

After making changes to any of these options, click the Save Preferences button.

5. Your Organization

From the Home  Your Organization page you can modify your organization’s Configuration  ] and menu:Organization Trusts[ and manage Cofiguration Channels distributed across an organization.

5.1. Configuration

On the Configuration  ] page modify your personal information  such as name  password  and title. To modify any of this information  make the changes in the appropriate text fields and click the menu:Update[ button at the bottom. image::home_your_organization_configuration.png[]

5.2. Organization Trusts

The Organization Trusts  ] page displays the trusts established with your organization (that is  the organization with which you  the logged-in user  are associated). The page also lists menu:Channels Shared[ , which refers to channels available to your organization via others in the established trusts. image::home_your_organization_trusts.png[]

You can filter the list of trusts by keyword using the Filter by Organization  ] text box and clicking menu:Go[ .

5.3. Configuration Channels

The Configuration Channels  ] page displays the channels which have been created and added using menu:Configuration[Configuration Channels .

From Home  Your Organization  Configuration Channels you can select which configuration channels should be applied across your organization. If there is more than one configuration channel selected you can specify the order of the channels. image::home_your_organization_custom.png[]

Procedure: Apply a Configuration Channel at the Organization Level

  1. Create a channel using Configuration  Configuration Channels or via the command line.

  2. Browse to Home  Your Organization  Configuration Channels .

  3. Use the search feature to locate a channel by name.

  4. Select the check box for the channel to be applied and click the Save Changes button. The save button will save the change to the database but will not apply the channel.

  5. Apply the channel by clicking the Apply button. The channel will be scheduled and applied to any systems included within the organization.
